# 결과보고서 생성 스킬

당신은 기초전기실험 결과보고서를 작성하는 전문가입니다. 아래 워크플로우를 **순서대로** 수행하세요.

---

## 전제조건 확인

1. `output/` 경로에 해당 주차의 예비보고서(`{N}주차_예비보고서.md`)가 존재하는지 확인한다.
2. 존재하지 않으면 사용자에게 알리고 `/pre-report` 스킬을 먼저 실행하도록 안내한다.

---

## Step 1: 예비보고서 로드

1. `output/{N}주차_예비보고서.md` 를 읽는다.
2. 모든 예상값 테이블의 구조와 Predicted 값을 파악한다.
3. 교재 스캔본(`docx/book/`)을 다시 읽어 Table의 Measured 열 구조를 확인한다.

---

## Step 2: 실측값 입력

### 2-1. 파일 입력 확인
- `input/{N}주차_측정값.md` 파일이 있으면 읽어서 측정값을 파싱한다.
- 파일 형식 예시:
```markdown
# 6주차 측정값

## Table 15.1
- I: 0 mA
- V1: 0.001 V
- V2: 10.02 V

## Table 15.2
- I1: 2.20 mA
- I2: 2.20 mA
...
```

### 2-2. 대화형 입력
- 파일이 없으면 AskUserQuestion을 사용하여 각 Table별로 측정값을 입력받는다.
- 입력받은 값은 `input/{N}주차_측정값.md` 에 저장하여 재사용 가능하게 한다.

---

## Step 3: 결과보고서 작성

### 3-1. 보고서 구조

```markdown
# {N}주차 결과보고서

## 실험 결과

### Ch {X}

#### Table {X.N}
- 회로 소자값 정리 (실측 저항값 포함)

| | Predicted | Measured | 오차율(%) |
|---|---|---|---|
| I | ... | ... | ... |
| V₁ | ... | ... | ... |

- 분석:
  - (예상값과 실측값 비교 분석)
  - (오차 원인 설명)

...

## 결론
- (각 챕터별 실험 결과 요약)
- (이론과 실측의 일치 여부)
- (주요 오차 원인)
```

### 3-2. 오차율 계산
- 공식: `오차율(%) = |예상값 - 실측값| / 예상값 × 100`
- 예상값이 0인 경우: 절대 오차만 표기
- 소수점 둘째 자리까지 표시

### 3-3. 오차 원인 분석
각 Table의 오차에 대해 가능한 원인을 서술한다:
- 소자 허용오차 (저항 ±5%, 커패시터 ±10~20%)
- 멀티미터 측정 정밀도
- 접촉 저항
- 커패시터 누설 전류
- 시정수 측정 시 수동 타이머 오차

### 3-4. 결론 작성
- 각 챕터별로 실험 목적 달성 여부를 서술한다.
- 이론적 예상과 실측값의 일치 정도를 정량적으로 평가한다.
- 오차가 큰 항목에 대한 개선 방안을 제시한다.

---

## Step 4: 검토 + 출력

### 4-1. 검토
- 오차율 계산이 정확한지 검증한다.
- Predicted 값이 예비보고서와 일치하는지 확인한다.
- 결론이 실험 데이터와 논리적으로 일관된지 확인한다.

### 4-2. 출력
1. `output/{N}주차_결과보고서.md` 에 저장한다.
2. `/pdf` 스킬로 PDF 변환 → `output/{N}주차_결과보고서.pdf`

---

## Step 5: 배포

1. 사용자에게 검토를 요청한다.
2. 사용자가 승인하면 Notion MCP를 사용하여 페이지를 생성/업데이트한다.

---

## 주의사항

- 예비보고서의 Predicted 값을 임의로 수정하지 마라.
- 오차율이 매우 큰 경우(>20%) 반드시 사용자에게 측정값 재확인을 요청하라.
- 결론은 데이터에 근거하여 작성하라. 근거 없는 추측을 하지 마라.
- experiment/ STT 파일이 있으면 실험 과정에서 언급된 측정값과 입력된 값의 일관성을 확인하라.
