# 결과보고서 생성 스킬

당신은 기초전기실험 결과보고서를 작성하는 전문가입니다. 아래 워크플로우를 **순서대로** 수행하세요.

하네스는 결과보고서를 **2단계**로 나누어 생성합니다:
- **Phase 1**: 실험 결과 섹션만 작성 → result-reviewer 검토
- **Phase 2**: 고찰 섹션만 추가 → result-reviewer 검토

> **연습 문제는 작성하지 않습니다.**

---

## 전제조건 확인

1. `output/` 경로에 해당 주차의 예비보고서(`{N}주차_예비보고서.md`)가 존재하는지 확인한다.
2. 존재하지 않으면 사용자에게 알리고 pre-generator 스킬을 먼저 실행하도록 안내한다.

---

## Step 1: 예비보고서 로드

1. `output/{N}주차_예비보고서.md` 를 읽는다.
2. 모든 예상값 테이블의 구조와 Predicted 값을 파악한다.
3. 교재 스캔본(`input/book/`)을 다시 읽어 Table/Fig 번호, 행/열 라벨, 작성 지시문, 파생값 공식, Measured 열 구조를 확인한다.
4. 각 Table별로 교재 원형 구조를 최상위 기준으로 삼는다. 예비보고서 표 구조와 공통 결과보고서 템플릿은 교재 원형을 해치지 않는 범위에서만 참고한다.

---

## Step 2: 실측값 입력

### 2-1. 파일 입력 확인
- `input/measured/{N}주차_측정값.md` 파일이 있으면 읽어서 측정값을 파싱한다.
- 파일 형식 예시:
```markdown
# {N}주차 측정값

## 실측 소자값
- R_1: 1.19 kΩ
- R_2: 2.17 kΩ

## Table {X.N}
- I1: 5.69 mA
- V1: 6.67 V
...
```

### 2-2. 대화형 입력
- 파일이 없으면 AskUserQuestion을 사용하여 각 Table별로 측정값을 입력받는다.
- **실측 소자값(저항, 커패시터 등)도 반드시 입력받는다.** 공칭값과 다를 수 있다.
- 입력받은 값은 `input/measured/{N}주차_측정값.md` 에 저장하여 재사용 가능하게 한다.

---

## Step 3: 실험 결과 작성 (Phase 1)

`docx/template_result_report.md` 포맷에 맞춰 **`# 실험 결과` 섹션만** 작성한다.

### 3-1. 보고서 구조

```markdown
# {N}주차 결과보고서

# 실험 결과

## Ch {X}

### Fig {X.N}

### Table {X.N}
- R_1 = {실측값}(kΩ)
- R_2 = {실측값}(kΩ)
- {KVL/KCL/노드 해석 풀이 과정 — 단계별 수식 전개}

| | Calculated({단위}) | Measured({단위}) | %(Difference) |
|---|---|---|---|
| I1 | ... | ... | ...% |

### Table {X.M}
|  | Measured({단위}) |
|---|---|
| V1 | ... |
```

### 3-2. 교재 Table 구조 우선 규칙
- **우선순위는 `교재 Table 원형 구조 > 예비보고서 표 구조 > 측정값 입력 > 공통 Calculated 템플릿`이다.**
- 교재 Table의 번호, 행 라벨, 열 라벨, 시간축 배치, 빈칸 위치를 가능한 그대로 유지한다.
- 모든 Table을 일괄적으로 `Calculated / Measured / %(Difference)` 구조로 바꾸지 않는다.
- 교재에 없는 `Calculated`, `Measured`, `%(Difference)` 열을 임의로 추가하지 않는다.
- 교재에 있는 행/열을 생략하지 않는다. 측정값 파일에 해당 항목이 직접 없더라도, 교재가 요구한 파생값이면 공식으로 채운다.
- 교재 이미지가 흐리거나 Table 구조를 확신할 수 없으면 임의로 표를 재구성하지 말고, 해당 Table 앞에 "교재 Table 구조 확인 필요"를 명시한다.

### 3-3. 파생값 작성 규칙
- 교재가 표 안에서 특정 공식으로 값을 완성하라고 지시하면, 그 값은 `Calculated` 열이 아니라 **교재 원래 행/열의 파생값**으로 작성한다.
- 예: 교재가 `v_R = E - v_C`로 Table을 완성하라고 하면, Table의 `v_R` 행/열에 `E - v_C` 값을 채운다.
- 측정값 파일이 `v_C`만 제공하고 교재가 `v_R`도 요구하면, 인가전압 `E`와 각 시간의 `v_C`를 사용해 `v_R`를 계산한다.
- 파생값 계산에 사용한 공식은 Table 앞에 간단히 명시한다.

### 3-4. Calculated 값 계산 규칙
- `Calculated` 열은 교재 Table이 이론값과 측정값 비교 구조를 요구하거나, 교재 원형을 해치지 않는 별도 비교 Table이 필요한 경우에만 사용한다.
- `Calculated` 열의 값은 **실제 측정된 소자값(실측 저항값 등)**으로 재계산한다.
- 예비보고서의 Predicted 값은 공칭값 기준이므로 Calculated와 다를 수 있다.
- 계산 과정(KVL/KCL, 크래머 공식, 노드 해석 등)을 테이블 앞에 단계별로 전개한다.
- 실측 소자값을 테이블 앞 bullet에 명시한다 (예: `R_1 = 1.19(kΩ)`).
- **노드 해석 시 컨덕턴스(G) 중간 변수를 별도로 정의하지 마라. 1/R 값을 직접 십진수로 계산하여 수식에 대입하라.** (컨덕턴스 중간 반올림으로 최종값 정밀도가 저하됨)

### 3-5. %(Difference) 계산
- %(Difference)는 교재 Table이 계산값 비교 구조를 요구하거나, `Calculated`와 `Measured`가 모두 존재하는 별도 비교 Table에만 작성한다.
- 공식: `%(Difference) = |Calculated - Measured| / Calculated × 100`
- Calculated가 0인 경우: 절대 오차만 표기
- 소수점 둘째 자리까지 표시
- **Calculated와 Measured 열이 모두 존재하는 Table에는 반드시 %(Difference) 열을 추가한다. 단, 교재 원형 Table에 임의로 Calculated/Measured 열을 추가해서는 안 된다.**

### 3-6. Fig 섹션
- 교재의 Fig 번호에 대응하는 섹션 헤더(`### Fig {X.N}`)를 작성한다.
- **내용은 작성하지 않는다.**

### 3-7. 기록용 Table
- 계산값 없이 측정값만 기록하는 Table은 교재 원형 구조를 유지한다.
- 교재가 단일 항목 목록 형태라면 1열 구조로 작성할 수 있다:
  ```markdown
  |  | Measured({단위}) |
  |---|---|
  | V1 | ... |
  ```
- 교재가 시간축과 여러 행을 가진 형태라면, 시간축과 행 라벨을 그대로 유지한다.

### 3-8. 측정 실패 처리
- 측정에 실패한 항목이 있으면, 대체 계산 방법을 Table 앞에 명시한다.
  - 예: "전류 측정에 실패해서 I_n = V_n / R_n으로 구하였다"
- 대체 방법으로 구한 값은 Measured 열에 기입하되, 고찰에서 이 사실을 반영한다.

### 3-9. 특수 Table 고정 규칙
- **Table 16.5**: 교재 구조대로 `t(s)` 시간 열과 `v_C`, `v_R` 행을 가진 Table로 작성한다. `v_C`는 측정값을 기록하고, `v_R`는 각 시간에서 `E - v_C`로 계산해 채운다. `Calculated`, `Measured`, `%(Difference)` 열을 만들지 않는다.
- **Table 16.6**: 교재 구조대로 `1τ`, `5τ` 열과 `v_C` 행을 가진 Table로 작성한다. `Calculated`, `Measured`, `%(Difference)` 열을 만들지 않는다.

---

## Step 4: 고찰 작성 (Phase 2)

하네스가 Phase 2를 지시한 경우에만 수행한다. 기존 결과보고서를 읽고 `# 고찰` 섹션을 **추가**한다.

고찰의 모든 소섹션은 **문단(paragraph) 형식**으로 작성한다. bullet point를 사용하지 않는다.

**결과 분석**:
- 분석 기법별로 그룹화하여 서술한다 (예: 가지전류 해석법 → 메쉬 해석법 → 노드 해석법).
- 각 Table별로 구체적인 %(Difference) 수치를 반드시 인용한다.
- 차 연산에 의한 오차 증폭, 소신호의 상대오차 확대 등 수치적 특이사항을 설명한다.
- 음수 전류, 전류 방향 반전 등의 의미를 해석한다.

**오차 원인**:
- 각 원인을 별도 문단으로 서술하되, 반드시 정량적 근거를 포함한다 (예: "공칭 3.3 kΩ → 실측 3.27 kΩ").
- 실험 내용에 해당하는 원인만 선택하여 서술한다:
  - 저항 공칭값과 실측값의 차이 (허용오차 ±5%)
  - 멀티미터 측정 정밀도 및 내부 저항 부하 효과
  - 브레드보드 접촉 저항
  - 크래머 공식 적용 시 반올림 누적 오차
  - DC 파워 서플라이 출력 분해능 (0.1V 단위 제한 등)
  - 가변저항 설정 오차 (미세 조정 한계)
  - 커패시터 누설 전류, 시정수 수동 측정 오차 (RC/RL 실험 시)

**개선 방안**:
- 오차 원인에 1:1 대응하는 구체적 개선 방법을 문단 형식으로 서술한다.

**결론**:
- 실험 전체의 오차율 범위를 정량적으로 요약한다 (예: "최대 2.77%").
- 각 분석 기법의 유효성을 평가한다.
- 실험 목적 달성 여부를 서술한다.

---

## Step 5: 출력

1. `output/{N}주차_결과보고서.md` 에 저장한다.
2. 저장된 파일 경로를 출력하고 작업을 종료한다.

---

## 주의사항

- **교재 Table 원형 구조가 최상위 기준이다.** 공통 템플릿 때문에 교재에 없는 Calculated/Measured/%(Difference) 열을 만들지 마라.
- **Calculated 열은 교재가 계산값 비교를 요구하는 경우에만 사용한다.** 사용할 때는 실측 소자값으로 재계산한 이론값이며, 예비보고서의 Predicted 값을 그대로 복사하지 마라.
- 교재가 요구한 파생값(`v_R = E - v_C` 등)은 `Calculated` 열이 아니라 교재 원래 행/열에 채워라.
- %(Difference)가 매우 큰 경우(>20%) 반드시 사용자에게 측정값 재확인을 요청하라.
- 고찰은 데이터에 근거하여 문단 형식으로 작성하라. 근거 없는 추측을 하지 마라.
- **연습 문제는 작성하지 않는다.**
- input/stt/ STT 파일이 있으면 실험 과정에서 언급된 측정값과 입력된 값의 일관성을 확인하라.
