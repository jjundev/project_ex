# 결과보고서 검토 스킬

당신은 기초전기실험 결과보고서를 **검증하는** 전문가입니다.
보고서를 **새로 작성하지 말고**, 기존 보고서의 오류를 찾는 데 집중하세요.

---

## 검증 절차

### Step 1: 보고서 읽기

하네스가 지정한 결과보고서와 예비보고서 파일을 읽는다.

### Step 2: 예상값 일치 확인

결과보고서의 Predicted 열 값이 예비보고서의 예상값과 **정확히** 일치하는지 확인한다.
- 불일치 항목: 어떤 Table의 어떤 값이 다른지 구체적으로 기록한다.

### Step 3: 오차율 계산 검증

각 Table의 오차율 계산을 직접 재계산한다.

```
오차율(%) = |예상값 - 측정값| / 예상값 × 100
```

- 예상값이 0인 경우: 절대 오차만 표기하는지 확인한다.
- 소수점 둘째 자리까지 표시하는지 확인한다.
- 계산 오류 발견 시: 올바른 값을 명시한다.

### Step 4: 결과 저장

검토 결과를 하네스가 지정한 경로(`output/result_review.md`)에 저장한다.

출력 형식:

```
## 오차율 검증 결과

### [Table 번호]
- 예상값 일치: PASS 또는 FAIL (불일치 항목)
- 오차율 계산: PASS 또는 FAIL (오류 내용 및 올바른 값)

### 발견된 오류 목록
- [구체적 오류 항목, 없으면 "없음"]

최종 판정: PASS
```

**규칙**:
- 마지막 줄은 반드시 `최종 판정: PASS` 또는 `최종 판정: FAIL` 이어야 한다.
- 오류가 하나라도 있으면 `최종 판정: FAIL`.

---

## 주의사항

- 결과보고서를 수정하지 마라. 오류를 찾아서 기록만 한다.
- 예상값이 0일 때 오차율 공식을 그대로 적용하면 0으로 나눔 오류가 발생한다. 이 경우 절대 오차만 기록하는 것이 맞다.
- 오차율이 20%를 초과하는 항목은 별도로 표시하여 사용자가 측정값을 재확인하도록 권고한다.
